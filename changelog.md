# Changelog

All notable changes to this project are documented in this file. Dates are in ISO 8601 format (UTC).

## [Unreleased]

- OS Child Lockdown: auto-creates passwordless `Child` standard user account, enforces strict machine-wide and per-user policies (TaskMgr, Regedit, CMD, Run, Control Panel, Windows Store, UAC max, etc.).
- Global CLI renamed from `dnslock` to `oslock`.
- Install directory moved from `C:\ProgramData\DNSGuard` to `C:\ProgramData\OSGuard`.
- Task names renamed from `DNS-Hijack-Guard` to `OS-Guard-Protection` and associated guardians.
- Added `ChildLogon` scheduled task that applies HKCU restrictions directly in the child's session at every logon.
- Added `Mount-ChildHive` / `Dismount-ChildHive` helper functions to load and edit the child's `NTUSER.DAT` offline.
- `New-LocalUser -NoPassword` used for passwordless child account creation with `net user /passwordchg:no /passwordreq:no` hardening.
- Added `Enable-OSLock` and `Disable-OSLock` functions alongside preserved `Enable-DNSLock` / `Disable-DNSLock`.
- Interactive menu now shows DNS status, OS child lockdown status, and installation integrity in a single unified dashboard.
- WMI event subscription renamed to `OSGuardWmiHealth`.
- Integrity hash registry key renamed from `PushConfigBackoffInterval` to `OSGuardIntegrity`.
- Added regression test suite in `tests/` folder with `syntax_check.ps1` and `test_os_lockdown.ps1`.
- **Stricter OS Lockdown** (2026-06-30T04:12:00Z): Windows Installer blocked (`DisableMSI=2`), USB storage disabled (`USBSTOR Start=4`), Windows Script Host disabled (`Enabled=0`), SmartScreen enforced (`Block` level), Fast User Switching disabled, Windows Update UI blocked for standard users, right-click context menu disabled, Folder Options hidden, taskbar changes blocked, printer add/remove blocked, and "This PC" hidden from desktop/start menu.
- **Admin-Approval Logout Shortcut** placed on the child's desktop. Clicking it triggers a UAC elevation prompt; the child cannot log out without an administrator entering credentials.
|- **Interactive TUI Category Status Grid** added: a compact two-column grid shows all 25+ lock categories (DNS, UAC, Store, Installer, USB, WSH, SmartScreen, Fast User Switching, Windows Update, Child Account, Task Manager, Registry Tools, CMD/Run, Control Panel, Wallpaper/Themes, AutoPlay, Admin Tools, Add/Remove Programs, Network UI, Password Change, Right-Click, Folder Options, Taskbar, Printers, This PC, Logout Shortcut, Background Service, Integrity) with [ENABLED] / [DISABLED] / [UNKNOWN] indicators so you can see every enabled and disabled category at a glance.
|- **Parent Mode, Game Request, AFK Timer & Shortcut Guardian** (2026-06-30T04:39:00Z): Parent Mode allows temporary password-protected admin unlock for installing software or viewing the child account. Three admin desktop shortcuts (`Parent Mode`, `Lock Now`, `Continue Parent Mode`) are created and guarded by the auto-heal system. Child has a `Request Game Install` shortcut that opens a simple game request dialog. A 1-minute AFK watcher (`OSGuard-ParentModeWatch`) monitors idle time and auto-triggers `oslock -LockNow` after 5 minutes of inactivity. Menu options `[7]` and `[8]` added. Default Parent Mode password is `admin123` (change via `oslock -SetParentPassword`). `Requests` directory created at `C:\ProgramData\OSGuard\Requests` with hardened ACLs.
- **Hardening Patch #1** (2026-06-30T04:50:00Z): Parent Mode password now uses a 16-byte random salt (Base64-encoded, stored in `OSGuardParentPasswordSalt`) alongside SHA256 to prevent offline brute-force. Minimum password length raised to 8 characters. `ParentModeWatch.ps1` is now embedded as a Base64 string inside the main script and rewritten fresh on every silent heal, with ACL hardened to SYSTEM only. Integrity checks now run immediately before `Enable-OSLock`, `Disable-OSLock`, and `Enter-ParentMode` (not just at menu render time). `Requests` directory ACL tightened: child user gets `WriteData, AppendData` only (no read/list/delete), while admins retain `ReadAndExecute`. WMI subscription health check added to `SilentLock` — re-registers `OSGuardWmiHealth` if filter/consumer/binding is missing. Task Scheduler service (`Schedule`) is monitored and auto-started if stopped during `SilentLock`.
- **Program Guardian Engine** (2026-06-30T06:06:00Z): New scheduled task `OSGuard-ProgramScanner` runs every 10 minutes as SYSTEM to scan the child profile for newly installed programs. Discovered program directories and `.lnk` shortcuts are hardened with ACLs so the child cannot modify, delete, or rename them, but can still execute. Windows built-in exploit tools (`notepad.exe`, `wordpad.exe`, `mspaint.exe`, `write.exe`, `explorer.exe`) are blocked via `DisallowRun` and `NoOpenWith` registry policies in the child's hive. The engine is registered during `Install-Persistence`, removed during `Uninstall-Persistence`, re-registered during `SilentLock`, and triggered immediately by `Enable-OSLock` and `-ProgramScan`.
|- **Comprehensive Exploit Block Expansion** (2026-06-30T08:16:00Z): `DisallowRun` registry list expanded from 5 to 50 entries covering PowerShell (`powershell.exe`, `pwsh.exe`), CMD (`cmd.exe`), Windows Script Host (`wscript.exe`, `cscript.exe`), HTML Application host (`mshta.exe`), download tools (`certutil.exe`, `bitsadmin.exe`, `curl.exe`), WMI command line (`wmic.exe`), COM/DLL execution (`regsvr32.exe`, `rundll32.exe`), installer (`msiexec.exe`), system configuration (`msconfig.exe`), management console (`mmc.exe`), UAC bypass vectors (`eventvwr.exe`, `fodhelper.exe`, `computerdefaults.exe`, `slui.exe`, `dccw.exe`, `xwizard.exe`), process termination (`taskkill.exe`), network tools (`ftp.exe`, `tftp.exe`, `telnet.exe`), file copy/permission tools (`robocopy.exe`, `takeown.exe`, `icacls.exe`), network commands (`net.exe`, `net1.exe`), task scheduling (`schtasks.exe`, `at.exe`), disk cleanup exploit (`cleanmgr.exe`), backup tool (`sdclt.exe`), system properties (`systempropertiesadvanced.exe`), Settings app (`ms-settings.exe`), Control Panel (`control.exe`), CPL files (`inetcpl.cpl`, `appwiz.cpl`), MSC files (`compmgmt.msc`, `diskmgmt.msc`, `devmgmt.msc`), and TaskMgr/Regedit/PerfMon (`taskmgr.exe`, `regedit.exe`, `perfmon.exe`). Additional Explorer policies added: `NoInternetOpenWith`, `NoSecurityTab`, `NoHardwareTab`, `NoManageMyComputerVerb`. `Scan-And-Harden-ChildPrograms` is now triggered immediately by `Enable-OSLock` and `Exit-ParentMode` so any software installed during Parent Mode is hardened the moment the lock is re-applied.
|- **Start Menu & Taskbar Lockdown + Parent Mode Window Guard** (2026-06-30T09:15:00Z): Added 20+ new per-user Explorer policies: `NoStartMenuPinnedList`, `NoStartMenuDragDrop`, `NoTrayContextMenu`, `NoMovingBands`, `NoCloseDragDropBands`, `NoBalloonTips`, `DisableContextMenusInStart`, `NoStartMenuNetworkPlaces`, `NoStartMenuEjectPC`, `NoStartMenuMyGames`, `NoStartMenuMyMusic`, `NoStartMenuMyPictures`, `NoStartMenuMyVideos`, `NoStartMenuDownloads`, `NoStartMenuDocuments`, `NoStartMenuRecordings`, `NoStartMenuHomegroup`, `NoStartMenuFavorites`, `NoStartMenuRecentDocs`, `NoStartMenuRun`, `NoStartMenuFind`, `NoStartMenuHelp`, `NoStartMenuLogoff`. Added machine-wide `DisableNotificationCenter` and `DisableWindowsConsumerFeatures` to block Action Center and suggested apps. New `Start-WindowGuard` and `Stop-WindowGuard` functions launch a hidden background PowerShell process during Parent Mode that polls every 5 seconds for new visible processes; whitelisted system processes are ignored. If a non-whitelisted window appears, an `InputBox` password prompt is shown using the same salted SHA256 hash. Three wrong passwords or Cancel triggers `oslock -LockNow` immediately. The guard is started in `Enter-ParentMode` and stopped in `Exit-ParentMode` and `Uninstall-Persistence`.
|- **Parent Mode Admin Tools + UI Refresh** (2026-06-30T09:40:00Z): `Enter-ParentMode` now restarts Windows Explorer after unlocking so the policy changes take effect immediately, and dynamically creates `Admin CMD.lnk` and `Admin PowerShell.lnk` on the admin desktop with the "Run as administrator" flag so the UAC prompt pops up for quick elevated terminal access. `Exit-ParentMode` removes these shortcuts before re-locking, then restarts Explorer again so the lock is visible immediately. The uninstaller also cleans up these shortcuts.
- **Edge-Only Browser Lockdown + Screen Time Engine** (2026-06-30T09:47:00Z): `DisallowRun` expanded to entries 51-58 blocking `chrome.exe`, `firefox.exe`, `brave.exe`, `opera.exe`, `vivaldi.exe`, `waterfox.exe`, `tor.exe`, `iexplore.exe` so only Microsoft Edge remains viable. Edge is deeply locked down via HKLM policies: `BookmarkBarEnabled=0`, `InPrivateModeAvailability=1`, `DeveloperToolsAvailability=2`, `DownloadRestrictions=3`, `SyncDisabled=1`, `PasswordManagerEnabled=0`, `URLBlocklist` (settings/flags/extensions/downloads/passwords/history/bookmarks), `ExtensionInstallBlocklist=*`, and more. New `ScreenTime.json` / `ScreenTimeTracker.json` config and tracker store daily activity hours, daily max minutes, and browser max minutes under `C:\ProgramData\OSGuard` with ACL hardened to SYSTEM only. `OSGuard-ScreenTime` scheduled task runs every minute as SYSTEM to enforce limits and terminate `msedge.exe` when time is exceeded. Child desktop receives `Browser Request.lnk` which opens `BrowserLauncher.ps1`; if no admin-granted allowance is active, the child sees a popup asking to contact the admin. Admin desktop receives `Grant Browser Time.lnk` which opens a password-protected dialog (`oslock -GrantBrowserTime`) requiring the Parent Mode password before setting session minutes (15/30/60/120). The timer is tamper-proof because the child cannot write the tracker file or run the admin dialog. New CLI flags: `-SetScreenTime`, `-ScreenTimeStatus`, `-GrantBrowserTime`, `-ScreenTimeEnforce`.
- **Category Status Grid Bug Fix** (2026-06-29T21:28:00Z): Fixed `NullArrayIndex` runtime error in `Show-CategoryGrid` (`new2_OS_lockdown.ps1:2591`). The `[ordered]@{}` `.Keys` property returns an `OrderedDictionaryKeyValueCollection` that does not support numeric indexing (`$Keys[$i]`). Changed `$Keys = $Categories.Keys` to `$Keys = @($Categories.Keys)` to cast the keys into a standard array before the two-column grid loop. This prevents the grid from crashing when rendering lock categories in the TUI.
- **Strict Mode Registry Fix** (2026-06-30T12:40:00Z): Fixed `PropertyNotFoundException` errors thrown by `Set-StrictMode -Version Latest` when registry properties are missing. Replaced all unsafe `(Get-ItemProperty -Path ... -Name ... -ErrorAction SilentlyContinue).Property` patterns with `Get-ItemProperty -Path ... -Name ... -ErrorAction SilentlyContinue | Select-Object -ExpandProperty ... -ErrorAction SilentlyContinue`, which safely returns `$null` instead of throwing. Applied across `Show-CategoryGrid`, `Show-HealthCheck`, `Enable-OSLock` verification, `Test-ParentPassword`, `Start-WindowGuard`, and all other registry read sites.
- **Category Grid [UNKNOWN] Fix** (2026-06-30T12:40:00Z): Fixed `Show-CategoryGrid` rendering `[UNKNOWN]` for all child-hive categories when the child account has never logged in (no `NTUSER.DAT` exists). Changed fallback values from `$null` to `$false` in the `else` block so they display as `[DISABLED]`. Added a `$HiveLoaded` boolean flag to track manual `reg.exe load` operations so `reg.exe unload` only runs when the script actually loaded the hive, preventing unnecessary unload attempts on live sessions. Changed the Child Account check from inline `Get-LocalUser` to `Get-ChildAccount` for consistency.
